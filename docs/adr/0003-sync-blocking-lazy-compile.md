# ADR-0003: 첫 요청 동기 블로킹 lazy 컴파일 + 파일별 락

상태: 승인 (2026-07-12)

## 맥락

캐시가 없는 `.ai.md`에 대한 첫 요청 시 LLM 컴파일은 30~90초가 걸린다.
그동안 요청을 어떻게 처리할지 정해야 한다.

## 결정

**동기 블로킹**: 요청을 잡고 있다가 컴파일 완료 시 결과를 그 요청에 바로 반환한다.

- nginx `proxy_read_timeout 300s`.
- 같은 파일에 대한 동시 첫 요청은 **파일별 `threading.Lock`으로 병합**하여
  LLM 호출이 중복으로 나가지 않게 한다. 락 획득 후 stale 여부를 재확인한다
  (double-checked locking).

기각한 대안: 202 + "컴파일 중" 로딩 페이지 + 폴링. 브라우저 UX는 좋지만
API 요청(동기 JSON 기대)에는 적용 불가라 경로가 갈라지고, 컴파일 상태 추적이
추가로 필요해 POC 복잡도가 눈에 띄게 오른다.

## 결과

- curl과 브라우저가 동일하게 동작한다. 구현이 가장 단순하다.
- 첫 방문자는 수십 초 대기한다 — 데모에서는 "AI가 지금 컴파일 중"이라는 연출로 수용.
