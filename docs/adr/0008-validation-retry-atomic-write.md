# ADR-0008: 검증 강도, 1회 자가 수정 재시도, 원자적 쓰기

상태: 승인 (2026-07-12)

## 맥락

"문법 검사 통과분만 dist에 덮어쓴다"(초안 4.2-4)의 검사 강도와
실패 시 재시도 정책을 정해야 한다.

## 결정

검증 강도:
- **API (`.py`)**: 2단계 — ① `ast.parse` 문법 검사, ② 격리된 모듈로 **실제
  import** + 모듈에 `app`(FastAPI 인스턴스) 존재 확인. 핫스왑(ADR-0004)이
  import 성공을 전제로 하므로 import 검증이 곧 배포 게이트다.
- **SPA (`.html`)**: 느슨하게 — 비어있지 않음 + `<html` 포함 + 마크다운
  코드펜스 제거 확인. 브라우저가 관용적으로 파싱하므로 엄격 검증은 실익이 없다.

재시도: 검증 실패 시 에러 메시지를 붙여 LLM에 수정을 **1회만** 재요청.
그래도 실패하면 포기 — 기존 캐시 유지 + 영어 에러 로그 1줄.
첫 요청인데 기존 캐시도 없으면 HTTP 502 + 간결한 JSON 에러 본문.
무한 재시도는 동기 블로킹(ADR-0003)과 결합하면 요청이 영원히 매달리므로 금지.

쓰기: 같은 디렉토리의 tmp 파일에 쓴 뒤 `os.replace`로 원자적 교체.
검증 통과 전에는 dist를 절대 건드리지 않는다.

## 결과

- 깨진 아티팩트가 dist에 존재할 수 없고, 실패가 가용성을 해치지 않는다.
