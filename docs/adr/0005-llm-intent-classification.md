# ADR-0005: LLM 사전 분류 + 키워드 폴백

상태: 승인 (2026-07-12)

## 맥락

명세가 SPA 타겟인지 REST API 타겟인지 판별해야 스캐폴딩 프롬프트를 고를 수 있다.
초안은 키워드 휴리스틱(HTML/UI/화면 → SPA, POST/GET/JSON/API → API)을 지시했으나,
"JSON을 fetch해서 화면에 그리는 대시보드" 같은 명세는 오분류된다.
오분류는 테트리스 명세가 FastAPI 코드로 컴파일되는 최악의 데모 실패를 낳는다.

## 결정

컴파일 전에 **초소형 LLM 분류 호출**(한 단어 답변 강제, temperature 0.0)로
`spa`/`api`를 판별한다. LLM 호출 실패 시 **키워드 휴리스틱으로 폴백**한다.

분류 결과는 따로 저장하지 않는다 — 산출물 확장자(`.html` vs `.py`)가
곧 분류의 영속 캐시다.

## 결과

- 어떤 명세가 와도 견고하고, 비용은 컴파일당 수십 토큰 1회로 무시할 수준.
- 폴백 덕에 분류 단계가 가용성의 단일 실패점이 되지 않는다.
- 초안 명세의 키워드 방식과 다르지만, "동작하는 POC" 기준에서 의도적으로 이탈.
