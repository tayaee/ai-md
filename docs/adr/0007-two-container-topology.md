# ADR-0007: 컨테이너 2개 — nginx + aimd-engine 단일 프로세스

상태: 승인 (2026-07-12)

## 맥락

초안은 "독립 빌드 워커"를 언급했다. 컴파일러를 별도 컨테이너로 분리하면
ADR-0003의 파일별 컴파일 락과 ADR-0004의 모듈 스왑을 프로세스 경계 너머로
공유해야 해서 파일락·Redis 같은 IPC가 필요해진다.

## 결정

컨테이너 2개: `nginx` + `aimd-engine`.

`aimd-engine` 안에서 단일 Python 프로세스가 ① uvicorn 호스트 앱,
② watchdog 감시 스레드, ③ MiniMax 컴파일러 모듈을 모두 품는다.
락과 레지스트리는 프로세스 메모리 안의 `threading.Lock`/dict로 끝난다.

"독립 빌드 워커"의 의도는 "요청 처리와 컴파일이 서로 블로킹하지 않는 것"으로
해석한다 — 프로세스 분리가 아니라 스레드/이벤트루프 분리로 달성한다
(컴파일은 `asyncio.to_thread`, 요청 처리는 이벤트 루프).

볼륨: `src/` → engine 읽기 전용, `dist/` → engine 쓰기 / nginx 읽기 전용.
engine 포트(8000)는 호스트에 바인딩하지 않는다.

## 결과

- IPC 없음, 가동부 최소. 프로덕션 스케일이 필요해지면 그때 3-컨테이너로 분리.
