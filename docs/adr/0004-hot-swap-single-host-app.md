# ADR-0004: 단일 호스트 앱 + 동적 서브앱 핫스왑

상태: 승인 (2026-07-12)

## 맥락

`dist/*.ai.md.py` 갱신 시 Uvicorn을 데드락·다운타임 없이 리로드해야 한다(초안 6장).

## 결정

Uvicorn은 **호스트 ASGI 앱 하나만** 상시 구동한다. 호스트가 `dist/*.ai.md.py`를
importlib로 동적 로드하여 각 모듈의 `app` 객체를 레지스트리 dict에 보관하고,
요청 시 ASGI scope의 `root_path`/`path`를 조정해 서브앱을 직접 호출한다.

재컴파일 완료 시: 새 파일을 **새 모듈 객체로 import → 성공한 경우에만
레지스트리 참조를 원자적으로 교체**. import 실패 시 기존 모듈이 그대로 산다.

기각한 대안:
- `uvicorn --reload`: 프로세스 전체 재시작 — in-flight 요청이 끊기고,
  깨진 파일이 떨어지면 전 API가 죽는다. 무중단 요구와 정면 충돌.
- 파일당 Uvicorn 프로세스 + 포트 분배: 격리는 좋지만 포트·프로세스 수명
  관리가 POC 범위를 넘는다.

## 결과

- 무중단 핫 리로드, 실패 시 기존 캐시 유지, 프로세스 재시작 없음 — 세 요구를
  "레지스트리 dict + 스왑 함수 하나"로 모두 충족.
- `GET /<name>.ai.md` → `/docs` 리다이렉트로 Swagger 계약(ADR-0001)이 자연스럽게 성립.
